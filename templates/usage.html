Semantic Debian

Usage

The webserver on rdf.debian.net uses content negotiation to return a 303 redirect to either an HTML+RDFa, Turtle or RDF/XML representation of the requested resource.

Content negotiation is based on the HTTP Accept header used when requesting a resource. The current implementation will return Turtle if text/turtle is present in the Accept header, then RDF/XML if application/rdf+xml is present in the Accept header and fall back to HTML+RDFa if neither are present in the Accept header.

Simple Examples

For some tools that are not specifically designed for the semantic web, like curl and wget, it is necessary to provide the Accept header manually. For other tools, like rapper and semantic web browser, it will not normally be required as these tools already expect content negotiation.

wget
wget -O - --header="Accept: text/turtle" http://rdf.debian.net/debian 2>/dev/null
curl
curl -L --header="Accept: application/rdf+xml" http://rdf.debian.net/project/direwolf
rapper
rapper http://rdf.debian.net/release/experimental
Q&D RDF Browser
http://graphite.ecs.soton.ac.uk/browser/?uri=http://rdf.debian.net/maintainer/irl%40fsfe.org
